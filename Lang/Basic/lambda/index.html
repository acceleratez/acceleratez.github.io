    Lambda 表达式

    Lambda表达式（通常称为 Lambda）是一种在被调用的位置或作为参数传递给函数的位置定义匿名函数对象（闭包）的简便方法。Lambda 通常用于封装传递给算法或异步函数的少量代码行。

    回顾Java方法

    Java中的方法分为实例方法和静态方法。而不论是实例方法，还是静态方法，在本质上都相当于是过程式语言的函数。不过Java的实例方法隐含地传入了一个this变量，即实例方法总是有一个隐含参数this。

    而函数式编程 (Functional Programming) 是把函数作为基本运算单元，函数可以作为变量，可以接收函数，还可以返回函数。历史上研究函数式编程的理论是Lambda演算，所以我们经常把支持函数式编程的编码风格称为Lambda表达式。

    Java中的Lambda表达式

    Lambda表达式是一种匿名函数，它没有名称，但有参数列表、函数主体、返回类型，可能还有一个可以抛出的异常列表。Lambda表达式的语法在Java中如下：

    1
2
3
4
    //Java
(parameters) -> expression

(parameters) -> { statements; }

    其中，parameters是参数列表，expression 或 { statements; } 是Lambda 表达式的主体。如果只有一个参数，可以省略括号；如果没有参数，也需要空括号。

    Lambda表达式的写法只需要写出方法定义，即参数列表、函数主体、返回类型和可能抛出的异常列表。

    参数类型可以省略，因为编译器可以自动推断。

    使用例1：

    1
2
3
4
    MathOperation addition = (int a, int b) -> a + b;

int result = addition.operation(10, 5);
System.out.println(result); // 15

    这里，MathOperation是一个函数式接口，其包含一个抽象方法operation。Lambda表达式(a, b) -> a + b实现了这一个抽象方法。

    使用例2：

    1
2
3
4
    List<String> names = Arrays.asList("peter", "anna", "mike", "xenia");

names.sort((a, b) -> b.compareTo(a));//sort方法，对列表进行排序
names.forEach(name -> System.out.println(name));//forEach方法接收一个Consumer接口的实现，这里使用Lambda表达式实现了Consumer接口。

    Why Lambda?

    在Java中，像Comparator, Runnable, Callable等接口内只定义了一个方法，被称作单方法接口。

    以接口Comparator和成员方法Arrays.sort()为例。我们可以使用传入匿名内部类的方式来编写，也可以使用Lambda表达式来编写。

    • 若使用匿名内部类的方法来编写：
    1
2
3
4
5
6
7
    String[] array = {"apple", "orange", "banana"};
Arrays.sort(array, new Comparator<String>() {
    @Override
    public int compare(String s1, String s2) {
        return s1.compareTo(s2);
    }
});
    • 若使用Lambda表达式的方法来编写：
    1
2
3
4
    String[] array = new String[] {"apple", "orange", "banana"};
Arrays.sort(array,
           (s1, s2) -> {return s1.compareTo(s2);}
           );

    可以看到，使用Lambda表达式的写法更加简单。

    从Java 8开始，Lambda表达式就被主要用作替换单方法接口。

    函数式接口

    • 如果一个接口只有一个抽象方法，那么该接口就是一个函数式接口。
    • 函数式接口的实例可以通过 Lambda 表达式、方法引用或者构造方法引用来创建。
    • 如果我们在某个接口上声明了 @FunctionalInterface 注解，那么编译器就会按照函数式接口的定义来要求该接口。
    • 如果某个接口只有一个抽象方法，但我们并没有给该接口声明 @FunctionalInterface 注解，那么编译器依旧会将该接口看作是函数式接口。

    在Java中， 四大核心函数式接口包括：

    1. Consumer（消费者）：接收一个输入参数并且不返回任何结果的操作。
    2. Supplier（供应者）：提供一个输出结果而不接受任何输入参数的操作。
    3. Function（函数）：接收一个输入参数，返回一个结果。
    4. Predicate（断言）：接收一个输入参数，返回一个布尔值结果。

// Comparator
@FunctionalInterface
public interface Comparator<T> {

    int compare(T o1, T o2); // Function 函数式接口

    boolean equals(Object obj);

    default Comparator<T> reversed() {
        return Collections.reverseOrder(this);
    }

    default Comparator<T> thenComparing(Comparator<? super T> other) {
        ...
    }
    ...
}

    注意到，虽然Comparator接口有很多方法，但只有一个抽象方法int compare(T o1, T o2)，其他的方法都是default方法或static方法。另外注意到boolean equals(Object obj)是Object定义的方法，不算在接口方法内。因此，Comparator也是一个FunctionalInterface。

    Python中的Lambda表达式

    Python使用 lambda 来创建匿名函数。其不需要使用 def 关键字来定义完整函数。它们没有函数名，只能通过赋值给变量或作为参数传递给其他函数来使用。

    Lambda 函数通常用于编写简单的、单行的函数，通常在需要函数作为参数传递的情况下使用。如map(), filter(), reduce() 等函数中。

    语法格式：

    1
    lambda arguments: expression

    使用例：

    1
2
3
4
5
6
7
8
    # 简单的匿名函数
x = lambda a, b, c : a + b + c
print(x(5, 6, 2)) # 13

# 配合内置函数使用
numbers = [1, 2, 3, 4]
result = map(lambda x: x ** 2, numbers)
print(list(result)) # [1, 4, 9, 16]

    C++中的Lambda表达式

    基本语法如下：

    1
    [capture list] (parameter list) -> return type { function body }
    其中：

    • capture list 是捕获列表，用于指定 Lambda表达式可以访问的外部变量，以及是按值还是按引用的方式访问。捕获列表可以为空，表示不访问任何外部变量，也可以使用默认捕获模式 & 或 = 来表示按引用或按值捕获所有外部变量，还可以混合使用具体的变量名和默认捕获模式来指定不同的捕获方式。
    • parameter list 是参数列表，用于表示 Lambda表达式的参数，可以为空，表示没有参数，也可以和普通函数一样指定参数的类型和名称，还可以在 C++14 中使用 auto 关键字来实现泛型参数。
    • return type 是返回值类型，用于指定 Lambda表达式的返回值类型，可以省略，表示由编译器根据函数体推导，也可以使用 -> 符号显式指定，还可以在 C++14 中使用 auto 关键字来实现泛型返回值。
    • function body 是函数体，用于表示 Lambda表达式的具体逻辑，可以是一条语句，也可以是多条语句，还可以在 C++14 中使用 constexpr 来实现编译期计算。

    使用例：
