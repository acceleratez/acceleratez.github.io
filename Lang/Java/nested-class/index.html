    嵌套类

    在Java中，对于嵌套类(Nested Class)的定义是：

    A nested class is any class whose declaration occurs within the body of another class or interface. A top level class is a class that is not a nested class.

    这里，一般把定义内部类的外围类成为包装类(Enclosing Class)或外部类(Outer Class)。

    如果根据NestedClass定义的位置，可以把嵌套类分为： - 成员嵌套类(Member Nested Class); - 局部嵌套类(Local Nested Class); - 匿名嵌套类(Anonymous Nested Class).

    大多数情况下，根据是否被static修饰，可以把成员嵌套类分为： - 静态嵌套类(Static Nested Class); - 非静态嵌套类(Non-static Nested Class).

    其中，非静态嵌套类即内部类(Inner Class). 其又被分为三种：

    1. 成员内部类(Member Inner Class);
    2. 局部内部类(Local Inner Class);
    3. 匿名内部类(Anonymous Inner Class).

    根据定义位置进行分类

    1. 成员嵌套类：成员嵌套类是作为外围类的一个成员存在的，可以访问外围类的所有成员，包括私有成员。其有EnclosingClass的属性。其可以使用public, protected, default, private四种访问修饰符修饰，也可以使用static, final等关键字修饰。

    2. 局部嵌套类：局部嵌套类是定义在外围类的方法里面的类，只能在方法内部使用，不能在方法外部使用。其有EnclosingClass和enclosingMethod两个属性。其可以使用static, final关键字修饰。

    3. 匿名嵌套类：匿名嵌套类是一个没有显式定义的类，其可以直接通过new方法创建类的实例。一般回调模式情况下使用的比较多。该种类不能使用任何关键字和访问控制符进行修饰。

    根据static修饰进行分类

    1. 静态嵌套类：就是使用static修饰的成员嵌套类；

    2. 非静态嵌套类：除上述之外的其他所有嵌套类的总称，即没有使用static定义的NestedClass。

      注意：InnerClass不能被定义成static，也不能有static方法初始化语句块。

      An inner class is a nested class that is not explicitly or implicitly declared static. Inner classes may not declare static initializers or member interfaces.

    InnerClass

    非静态嵌套类可以分为以下三种：

    1. 成员内部类：没有使用static修饰的成员内部类。
    2. 局部内部类：定义在方法里面的内部类，方法可以是static的也可以是非static的，也可以是构造器方法。
    3. 匿名内部类：定义在方法里面匿名类，方法可以是static的也可以是非static的。

    嵌套类访问规则

    StaticNestedClass访问规则

    • 使用static修饰的嵌套类，只能访问外部类的非静态变量。
    • 对于public的静态嵌套类可以使用new OuterClass.InnerClass()的方式来创建。
    • 对于default的静态嵌套类可以在同一包名下，使用new OuterClass.InnerClass()的方式创建。
    • 特别地，对于静态成员类，其可以使用访问控制符和static修饰，可以是abstract抽象类。

    public class StaticNestedClass{

    // 私有局部
    private int i = 0;
    // 静态
    public static int j = 0;
    // 不变值
    private final int k = 0;
    // static final
    private static final int m = 0;

    // 静态嵌套内，这里不是innerclass,可以直接new出来
    public static class PublicNestedClass {
        private void test1() {
            // System.out.println(i); 非innerClass不能访问enclosing类的非static属性
            System.out.println(j);
            System.out.println(m);
            // System.out.println(k); 非innerClass不能访问enclosing类的非static属性
        }  
        // 可以定义static方法
        private static void test2() {

        }
    }
    // 静态嵌套内，这里不是innerclass，由于是私有的，不可以直接new出来
    private static class PrivateNestedClass {

    }
}

    InnerClass访问规则

    内部成员类

    • 内部成员类可以访问外部类的所有实例属性、静态属性。因为内部成员类持有一个外部对象的引用，内部类的实例可以对外部类的实例属性进行修改。
    • 对于public的内部类可以使用new OuterClass().new InnerClass()的方式来创建。（外部类实例.new 内部类()）
    • 调用内部类的构造器的时候，会把当前创建的内部类对象实例中持有的外部对象引用赋值为当前创建内部类的外部类实例。
    • 内部成员类可以是使用访问控制符，可以定义为final，也可以是抽象类。

    public class MemberInnerClass{
    // 私有局部
    private int i = 0;
    // 静态
    static int j = 0;
    // 不变值
    private final int k = 0;
    // static final
    private static final int m = 0;

    public class PublicMemberInnerClass{
        // 可以访问外部类的所有属性
        private void test1(){
            System.out.println(i);
            System.out.println(j);
            System.out.println(k);
            System.out.println(m);
        }

        public MemberInnerClass getOuterClass(){
            return MemberInnerClass.this;
        }

        // 不可以定义static方法
        // private static void test2() {
        // }
    }

    // 私有内部类，外部不能访问
    private class PrivateMemberInnerClass{

    }

    // 公开局部类，外部可以访问和创建，但是只能通过getOuterClass实例创建
    class DefaultMemberInnerClass{
        public MemberInnerClass getOuterClass(){
            return MemberInnerClass.this;
        }
    }
}

    局部成员类

    • 局部类 定义在类方法里面。这个方法既可以是静态方法，也可以是实例方法，也可以是构造器方法或者静态初始化语句块。
    • 局部类可以定义在一个static上下文里面和非static上下文里面。局部类不能有访问控制符(如public等)，可以是抽象的，也可以定义为final。
    • 定义在static上下文里面的局部成员类可以访问类的静态属性。如果定义在静态方法里面的局部类，还可以访问里面定义的final变量。在static上下文定义的局部类，没有指向父类实例变量的引用，因为static方法不属于类的实例，属于类本身。而且局部类不能在外部进行创建，只能在方法调用的时候进行创建。

    匿名内部类

    • 匿名内部类也是定义在方法里面，并且，其和局部类访问规则一样，只不过内部类显式的定义了一个类，然后通过new的方式创建这个局部类的实例，而匿名类直接new一个类实例，没有定义这个类。

    • 匿名类最常见的方式就是回调模式的使用，通过默认实现一个接口创建一个匿名类，然后new这个匿名类的实例。

    嵌套类的层次

    • 嵌套类可以有多层嵌套，即一个类里面可以有多个嵌套类，而这些嵌套类里面还可以有嵌套类。
    • 对于成员类，内部嵌套类可以表示为A.B的形式，其中，A是外部类，B是内部类。如果在B中定义了一个内部类C，那么C的表示形式为A.B.C。
    • 对于局部内部类，其表示形式为A.1B的形式，其中，A是外部类，1B是第一个局部内部类。如果在不同的方法里面定义了同名的局部类B，编译器是可以编译通过的，那么定义的第二个局部类B可以表示为A.2B。但是，如果在同一个方法中定义了两个同名的局部类，那么编译器会报错。
    • 对于匿名内部类，匿名类可以表示为A.N的形式，其中N为自然数，代表程序里面的第N个匿名类。

    嵌套类的应用

    在Java中大量使用了嵌套类。比如在Map类里面定义了一个Entry的抽象内部类，所以，在遍历map的时候，一般可以使用Map.Entry的方式来遍历map。
